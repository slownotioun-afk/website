# hamit.org — Website Template

A minimal, fast, self-hosted personal website running on a Cloudflare Tunnel
from a Proxmox LXC container. No port-forwarding required.

Design philosophy: load fast, be readable, stay out of the way.

## Project Structure

```
hamit-website/
├── src/
│   └── server.ts          # Express server (TypeScript)
├── public/
```

## Quick Start (local)

Requirements: Node.js >= 22

```bash
# 1. Install dependencies
npm install

# 2. Start in dev mode (ts-node, no build step)
npm run dev

# 3. Open http://localhost:3000
```

For production:

```bash
npm run build    # compiles TypeScript → dist/
npm start        # runs dist/server.js
```

## Deployment to Proxmox LXC

### 1. Create the LXC container

In the Proxmox web UI or via CLI:

```bash
# Example: Debian 12, 512 MB RAM, 4 GB disk
# NOTE: replace the template filename with the latest available in your Proxmox host.
# Run `pveam update && pveam available --section system | grep debian` to find the current name.
pct create 100 local:vztmpl/debian-12-standard_12.10-1_amd64.tar.zst \
  --hostname hamit-web \
  --memory 512 \
  --swap 512 \
  --rootfs local-lvm:4 \
  --net0 name=eth0,bridge=vmbr0,ip=dhcp \
  --unprivileged 1 \
  --start 1
```

### 2. Install Node.js inside the LXC

```bash
pct enter 100          # or: ssh root@<LXC-IP>

# Install Node.js 22 LTS via NodeSource
apt update && apt install -y curl
curl -fsSL https://deb.nodesource.com/setup_22.x | bash -
apt install -y nodejs

node --version   # should print v22.x.x
```

### 3. Upload and build the project

From your machine:

```bash
# Copy project files to the LXC
scp -r ./hamit-website root@<LXC-IP>:/opt/hamit-website
```

Inside the LXC:

```bash
cd /opt/hamit-website
npm install --omit=dev        # skip devDependencies in production
npm run build                 # compile TypeScript
```

### 4. Install the systemd service

```bash
cp /opt/hamit-website/hamit-website.service /etc/systemd/system/
systemctl daemon-reload
systemctl enable hamit-website
systemctl start hamit-website
systemctl status hamit-website   # should show: active (running)
```

Check logs at any time:

```bash
journalctl -u hamit-website -f
```

## Cloudflare Tunnel Setup

No port-forwarding, no exposed ports. The tunnel runs inside the LXC and
connects outbound to Cloudflare's edge.

### 1. Install cloudflared inside the LXC

```bash
# Download the latest release for amd64
curl -L https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64.deb \
     -o /tmp/cloudflared.deb
apt install -y /tmp/cloudflared.deb

cloudflared --version
```

### 2. Authenticate cloudflared with your Cloudflare account

```bash
cloudflared tunnel login
# This opens a browser URL — open it on your desktop, authorise hamit.org
# A certificate is saved to ~/.cloudflared/cert.pem
```

### 3. Create the tunnel

```bash
cloudflared tunnel create hamit-tunnel
# Outputs something like:
#   Created tunnel hamit-tunnel with id: xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
#   Credentials file saved to: /root/.cloudflared/xxxxxxxx-...json
```

Note the tunnel UUID — you need it in the next step.

### 4. Configure the tunnel

```bash
mkdir -p ~/.cloudflared
cp /opt/hamit-website/cloudflared-config.yml ~/.cloudflared/config.yml
nano ~/.cloudflared/config.yml
# Replace REPLACE-WITH-YOUR-TUNNEL-UUID with the UUID from step 3
```

### 5. Add DNS records via cloudflared

```bash
# Point hamit.org → tunnel
cloudflared tunnel route dns hamit-tunnel hamit.org

# Point www.hamit.org → tunnel (optional)
cloudflared tunnel route dns hamit-tunnel www.hamit.org
```

This creates CNAME records in your Cloudflare DNS automatically.

### 6. Run the tunnel as a system service

```bash
cloudflared service install
systemctl enable cloudflared
systemctl start cloudflared
systemctl status cloudflared
```

### 7. Verify

```bash
# From inside the LXC
curl -I http://localhost:3000        # should return HTTP 200

# From your desktop
curl -I https://hamit.org           # should return HTTP 200 (via tunnel)
```

### Tunnel troubleshooting

```bash
journalctl -u cloudflared -f                    # live tunnel logs
cloudflared tunnel info hamit-tunnel            # tunnel status
cloudflared tunnel list                         # all tunnels
cloudflared tunnel delete hamit-tunnel          # delete if starting over
```

## Customising the Site

All user-facing content is in `public/index.html`.
Search for `REPLACE` — every placeholder is marked.

| What to change                    | Where                               |
|-----------------------------------|-------------------------------------|
| Your name / bio                   | `<h1>` and `<p>` below it           |
| Navigation links                  | `<nav>` and `<footer>`              |
| Projects list                     | `<section>` with id `projects-heading` |
| Writing / blog list               | `<section>` with id `writing-heading`  |
| Social links                      | `<section>` with id `elsewhere-heading` |
| Email address                     | Both `<nav>` and `<footer>`         |
| Page title / SEO description      | `<head>` meta tags                  |
| Colour scheme                     | `:root` CSS variables at top of `<style>` |
| Favicon letter                    | `public/icon.svg`                   |

### Colour scheme

Edit the `:root` block in `public/index.html`:

```css
:root {
  --bg:    #fafaf8;   /* page background */
  --fg:    #1a1a1a;   /* main text        */
  --muted: #555;      /* secondary text   */
  --link:  #0055cc;   /* link colour      */
  --accent:#0055cc;   /* borders / hr     */
}
```

Dark mode is included but commented out — uncomment the
`@media (prefers-color-scheme: dark)` block to enable it.

## Adding Pages

### 1. Create the HTML file

Copy `public/index.html` to e.g. `public/about.html` and edit the content.
The `<style>` block can be extracted into `public/style.css` and linked from
all pages to avoid duplication.

### 2. Add a route in the server

In `src/server.ts`, add:

```typescript
app.get("/about", (_req, res) => {
  res.sendFile(path.join(__dirname, "..", "public", "about.html"));
});
```

Then rebuild and restart:

```bash
npm run build
sudo systemctl restart hamit-website
```

### Tip: shared CSS file

Create `public/style.css`, move the `<style>` content there, then in each
HTML file replace the `<style>` block with:

```html
<link rel="stylesheet" href="/style.css" />
```

## Further Improvements

These are all optional. The site works fine without any of them.

### Performance

- **Brotli compression** — replace the `compression` npm package with
  `shrink-ray-current` for Brotli + gzip. Or pre-compress static files
  with `gzip` / `brotli` CLI tools and serve them directly.

- **Static site generation** — if you never need server-side logic, remove
  Express entirely and serve the `public/` folder with `cloudflared`'s
  built-in static file server:
  ```yaml
  # in config.yml
  - hostname: hamit.org
    service: file:///opt/hamit-website/public
  ```
  Zero Node.js required.

- **HTTP/2 push** — Cloudflare automatically upgrades to HTTP/2 and HTTP/3
  (QUIC) for all traffic. Nothing to configure.

### Content management

- **Markdown blog posts** — add the `marked` npm package and render `.md`
  files as HTML at runtime. Example:
  ```typescript
  import { marked } from "marked";
  app.get("/writing/:slug", (req, res) => {
    const md = fs.readFileSync(`content/${req.params.slug}.md`, "utf8");
    res.send(wrapInLayout(marked(md)));
  });
  ```
  Resources: https://github.com/markedjs/marked

- **Static site generator** — for larger sites, consider:
  - **Astro** (https://astro.build) — ships zero JS by default, very fast
  - **Eleventy** (https://11ty.dev) — simple, no framework lock-in
  - **Hugo** (https://gohugo.io) — single binary, extremely fast builds

### Monitoring

- **Uptime Kuma** — self-hosted uptime monitor, runs perfectly in another
  LXC container. Checks `/health` on your site.
  https://github.com/louislam/uptime-kuma

- **Cloudflare Analytics** — free, privacy-friendly, no tracking script
  needed. Enable in the Cloudflare dashboard under your domain → Analytics.

- **Grafana + Prometheus** — overkill for a personal site, but if you're
  already running it on Proxmox, add a scrape target for the `/health`
  endpoint.

### Security

- **Cloudflare WAF** — free tier includes basic WAF rules. Enable in the
  Cloudflare dashboard → Security → WAF.

- **Rate limiting** — add the `express-rate-limit` package to the server:
  ```bash
  npm install express-rate-limit
  ```
  ```typescript
  import rateLimit from "express-rate-limit";
  app.use(rateLimit({ windowMs: 60_000, max: 100 }));
  ```

- **Access control** — use Cloudflare Access (free for up to 50 users) to
  put a login wall in front of any subdomain (e.g. admin.hamit.org).
  https://developers.cloudflare.com/cloudflare-one/policies/access/

### Multiple services on one tunnel

Add more entries to `cloudflared-config.yml`:

```yaml
ingress:
  - hostname: hamit.org
    service: http://localhost:3000
  - hostname: git.hamit.org
    service: http://localhost:3001   # e.g. Gitea
  - hostname: files.hamit.org
    service: http://localhost:3002   # e.g. Filebrowser
  - service: http_status:404
```

Then add DNS records:

```bash
cloudflared tunnel route dns hamit-tunnel git.hamit.org
cloudflared tunnel route dns hamit-tunnel files.hamit.org
```
